# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Ghost Signal is a collection of browser audio themes — synthesized UI sound palettes built entirely with the Web Audio API. Each theme is a self-contained ES module (`sounds.js`) loaded dynamically by a shared demo page (`demo.html?theme=<name>`). Zero dependencies, zero build step, zero tooling.

## Commands

There are **no build, lint, or test commands**. Verification is manual: open `demo.html?theme=<theme-name>` in a browser.

**WAV export** (optional, requires Puppeteer):
```
npm install --no-save puppeteer
node export-wav.js <theme-name>
```

**Landing page** is auto-generated by `.github/generate-index.sh` on push to `master` via GitHub Actions.

## Architecture

- **`demo.html`** — Single shared UI page. Reads `?theme=<name>` param, dynamically imports `<name>/sounds.js`. Contains all CSS inline. All event handlers are inline (`onclick`, `onmouseenter`).
- **`<theme-name>/sounds.js`** — ES module exporting `{ meta, createSounds }`. The `meta` object defines UI metadata (name, subtitle, 9 color hex values, 16 sound labels). The `createSounds(ctx, noiseBuffer)` factory returns an object of 17 sound functions.
- **`<theme-name>/<theme-name>.md`** — Design spec defining mood, colors, Sonic DNA, and synthesis recipes for all 16 sounds.
- **`<theme-name>/index.html`** — Thin redirect to `../demo.html?theme=<name>`.
- **`TEMPLATE.md` / `TEMPLATE.js`** — Fill-in-the-blanks templates for new themes.
- **`export-wav.js`** — Headless Puppeteer script rendering themes to WAV files via OfflineAudioContext.

## Agent Skills

Skills are located in `.agents/skills/`. Each subdirectory contains a `SKILL.md` with a complete workflow for a specific task. Read the relevant skill file before executing that workflow.

- **`create-sound-theme`** — Full pipeline for creating a new theme from an image or visual reference. Covers image analysis, sonic metaphor mapping, color extraction, cover image generation via NotebookLM, and all file creation steps.

## Coding Guidelines

**Read `AGENTS.md` before making changes.** It contains the authoritative coding style, Web Audio patterns, Sonic DNA differentiation rules, and the full theme creation workflow.

Key points:
- 2-space indent, semicolons always, single quotes in JS, double quotes in HTML
- `function()` expressions for sound properties (not arrows), arrows only in callbacks
- `const now = ctx.currentTime` is always the first line in every sound function
- Short node names: `osc`, `bp`, `lp`, `hp`, `nSrc`, `nG`, `sG`, `bG`
- Chained `.connect()` calls, explicit `.stop()` for cleanup
- 9 required CSS color variables: `--accent`, `--accent2`, `--danger`, `--bg`, `--surface`, `--surface2`, `--border`, `--text`, `--text-dim`

## New Theme Checklist

1. Compare planned Sonic DNA against the table in `AGENTS.md` — must differ in at least 3 of 5 columns from every existing theme
2. Create `<theme-name>/` directory (kebab-case)
3. Fill `TEMPLATE.md` → `<theme-name>/<theme-name>.md`
4. Implement `TEMPLATE.js` → `<theme-name>/sounds.js` with all 17 sound functions
5. Create redirect `<theme-name>/index.html`
6. Web Audio API only — no sample files, no external libraries

## Git Conventions

- Imperative mood: "Add …", "Fix …", "Update …"
- Single-line message with em-dash detail: `Add Neon Pulse theme — PWM square with logic gate shimmer`
- No conventional commit prefixes (`feat:`, `fix:`)
- Commit only theme-related files — no generated files or tooling artifacts
